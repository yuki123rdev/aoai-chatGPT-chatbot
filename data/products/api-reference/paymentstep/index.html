expirationTime
Numeric Symbol [ 23 .. 23 ]

Expiration Time
Expiration time for the payment.

You can restrict the payment to be completed before this time.

Once exceeded the time, PayStep shows session timeout error page.

ex) expirationTime="2018-01-01 10:20:30.123"

if the current time on server side is "2018-01-01 10:20:31.123", we show session timeout

if the current time on server side is "2018-01-01 10:20:30.123" or less (considering also the tolerance of timestamp field), we process the transaction.

If the expiration time is not set and if the page is inactive for 30 minutes ,session will be expired. (the timer will be restart if users refresh the browser or process to another page), .

If session is timeout, there following error message will be shown

maxRedeemPoint
Numeric [ 1 .. 9 ]

MaxRedeemPoint
The maxRedeemPoint feature can be use to limit the max point the user can use for redeem during the payment.

A typical usage is when service needs to force card payment to be processed as 3DS.

Scenario:

  • User buys a product for 1,000 JPY
  • Service create PayStep request with grossAmount = 1000,maxRedeemPoint = 90,withThreeDSecure = true
  • User logins and PayStep shows the payment page which allows users to:
    1. Pay the full amount with card (3DS) or
    2. Pay the full amount with Point (assuming that user has enough balance to pay full amount)
    3. Pay with Point (minRedeemPoint DB setting on PayStep ~ up to maxRedeemPoint/Point Balance when balance is less then maxRedeemPoint) and card
      Users current balance(Point)maxRedeemPointcard amountpoint amountSampleNote
      0901,000-1,000 card + 0 point = 1,000JPY
      5090950 ~ 9991 ~ 50950 card + 50 point = 1,000JPYLimited by the user''s balance
      8990911 ~ 9991 ~ 89999 card + 1 point = 1,000JPYLimited by the user''s balance
      9090910 ~ 9991 ~ 90910 card + 90 point = 1,000JPY
      91 or more90910 ~ 9991 ~ 90910 card + 90 point = 1,000JPYUser cannot redeem more than maxRedeemPoint even when the balance is higher
    4. Screen Error
      When trying to input greater than max point limitation. In this case the maxPointRedeem is 999.

Please be aware that minimum point is defined during the account configuration (PayStep DB) and PaySet rejects the request when maxRedeemPoint < minimum point.

Used by fraud prevention : No

object

Grant

rate
Numeric [ 1 .. 2 ]

Rate
Will be used to calculate granted Rakuten Super Points to shopper.

The rate should not be zero and have ”decimal point”. It's accepted from 1 to 99.

granted points = (gross amount - used coupon amount) * rate / 100

ex) gross amount = 1,000, used coupon amount = 100, rate =1, then granted points = 9

(*) granted points will be rounded down "decimal point".

(*) You need to set either "points" or "rate". If you set both "points" and "rate", "rate" will be used.
