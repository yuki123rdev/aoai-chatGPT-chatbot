Info Vault (1.0.0)

Payment Gateway as one of the Rakuten Payment Platform products enables you to use various payment features via our APIs.

# API type Details
1 File/V1/Add PayVault offers a set of comprehensive API's in order to store information securely into the vault. InfoVault Add File API allows you to store image files into the Vault. Whenever an image is uploaded into the vault, it returns a unique image token. This token can be used to download the respective image later.
2 File/V1/Delete PayVault offers a set of comprehensive API's in order to store information securely into the vault. InfoVault Delete File API allows you to delete metadata associated with a file stored in the Vault. To delete this metadata, the file token or the request id can be used.
Services belongs to one group and they can delete sensitive information if it is same group.
3 File/V1/Get PayVault offers a set of comprehensive API's in order to store information securely into the vault. InfoVault Get File API allows you to retrieve metadata associated with a file stored in the Vault. To retrieve this metadata, the file token or the request id can be used.
Services belongs to one group and they can redeem sensitive information if it is same group.
4 File/V1/Redeem PayVault offers a set of comprehensive API's in order to store information securely into the vault. InfoVault Download File API allows you to retrieve stored images from the Vault. Whenever an image is uploaded into the vault, it returns a unique image token. This token should be used to download the respective image.
Services belongs to one group and they can redeem sensitive information if it is same group.
5 Text/V1/Add Add Text API is used to register sensitive information like mynumber, email address in InfoVault system. This API generates a correspondent token which is then returned to the appropriate service.
Services' system receives sensitive information from user and send it to InfoVault. Services store tokens instead of sensitive information in their system.
This API can be called only from services' system.
6 Text/V1/Redeem Redeem Text API is used to get sensitive information from PayVault system by textToken or requestId.
Services belongs to one group and they can redeem sensitive information if it is same group.

Character code

Character code Remarks
UTF-8 Both request and response are UTF-8

File/V1/Add

PayVault offers a set of comprehensive API's in order to store information securely into the vault. InfoVault Add File API allows you to store image files into the Vault. Whenever an image is uploaded into the vault, it returns a unique image token. This token can be used to download the respective image later.


Please do not upload files like credit card image, etc in the InfoVault. In case if you want to register credit card information, please use /api/pv/Add/V1/Add.

Please check out more details on our FAQ page

Endpoint

Environment URL Remarks
Staging https://stg-infovault.global.rakuten.com/api/pv/File/V1/Add
Production https://infovault.global.rakuten.com/api/pv/File/V1/Add

Request Headers

Header Value
Content-Type multipart/form-data
User-Agent {UserAgent}

Or

{UserAgent};serviceId={serviceId};transactionId={transactionId}		 The additional values for User-Agent header are not required for processing request at server side. However, we strongly recommend to add these two values to User-Agent header. The reason is, in case the request fails to reach PayVault application server due to network connectivity issues, those values allows PayVault Team to track what service and what transaction was impacted by the issue.

・The serviceId is serviceId PayVault issued and shared with you previously. It can be used to investigate impact by serviceId.
・The transactionId is an unique key for each request sent to PayVault. It can be used to investigate where request failed exactly.
E.g.

Java/1.8.0_91

Java/1.8.0_91;serviceId=all-demo;transactionId=874fer222

File
Requirements

  • Image file
  • MIME type among
    • image/jpeg
    • image/png
    • application/pdf.
  • Within 5MB per file
  • File name should be Alphanumeric and may contain '_' or '.' or '-' characters.
  • File name should be within 0 - 100 chars. If filename is longer than 100, "invalid_request_parameter" will be returned.
To start with, we accept limited file formats to be stored in the InfoVault. Please let us know in case your service needs any additional file formats to be supported.

Acceptable file name examples could be

file.png

file001.png

file_underscore.jpg

file-hyphen.png

file.dot.jpg

file_underscore.dot.jpg
