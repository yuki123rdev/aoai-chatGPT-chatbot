/api/pv/File/V1/Add​

Request Parameters​

ParameterRequiredDescriptionRemarks
request_infoYBase64 encoded string of request JSONThe sample of request JSON is below.
signatureYHMAC_SHA256(key, request_json)Please refer to Authentication...
key_versionYVersion of a previously shared key
fileYRequirements

・Image file
・MIME type among
 ・image/jpeg
 ・image/png
 ・application/pdf
・Within 5MB per file
・File name should be Alphanumeric and may contain '_' or '.' or '-' characters.
・File name should be within 0 - 100 chars. If filename is longer than 100, "invalid_request_parameter" will be returned.		To start with, we accept limited file formats to be stored in the InfoVault. Please let us know in case your service needs any additional file formats to be supported.

Acceptable file name examples could be

file.png

file001.png

file_underscore.jpg

file-hyphen.png

file.dot.jpg

file_underscore.dot.jpg

Example Request and Response​

Example Request with curl​

This curl command cannot be used directly. Because the file must exist on where you run curl command. And the requestId and file's name must be different by each request within one serviceId.
 1. Make a file on where you run culr command.
 2. Replace "stg-test-001.png" with the name of file you made.
 3. Go to https://payvault-tool-testing-dev.ap-northeast-1.elasticbeanstalk.com/testing/infovault/api/api_test
 4. Copy the example request JSON above to Request Data on the page
 5. Change "requestId" to a new one.
 6. Change HMAC Key to your service HMAC key.
 7. Replace request_info and signature in the curl below with the ones on the page

curl -i -X POST -H "Content-Type: multipart/form-data" \
-F "request_info=ewogICAgInNlcnZpY2VJZCI6ImRlbW80OCIsCiAgICAidGltZXN0YW1wIjoiMjAxNi0wOC0wOSAxNjowMDowMC4xMjMiLAogICAgInJlcXVlc3RJZCI6ICJGRFNGU0ZTMjNSRVJFUkVTRDMzIiwKICAgICJjdXN0b20iOnsic29tZS1rZXkiOiJmb3ItdGVzdGluZyJ9Cn0" \
-F "signature=aed1c87d6d3134e459c6eb2242ca52c105014d7605da78c37c7e7756168e400e" \
-F "key_version=2" \
-F "file=@stg-test-001.png" \
https://stg-infovault.global.rakuten.com/api/pv/File/V1/Add

Example Response of Success​

If the request parameters are correct and the given service has valid permissions to access the file, the target file will be streamed back to the calling component (Browser, Rest Client, etc). In case of browser, the file will be downloaded. The JSON of response will not be returned in this case.
