/Payment/V1/CancelOrRefund​

Request Data​

Example of Request Data​

HTTP Method: POST​
Request Headers​
HeaderValueRemarks
Content-Typeapplication/x-www-form-urlencoded

Response Data​

If the Authorized/Captured amount is 0 (e.g. The transaction was authorized/captured with the card amount 0), we do not proceed with the cancellation of the transaction with the PSP, and this API will return a success response.

Note that the CancelOrRefund API's response will contain the previous transaction reference, as we don't process the cancellation with the PSP.

If the transaction has the information about card+Rakuten Points (Card amount 0 + Points 1000), we cancel the used Rakuten Points and return the response.

        "subscriptionId" : "sub_001"
  }
}		(*8) When the subscription payment method is used to Authorize or AuthorizeAndCapture, the reference will contain subscriptionInfo along with the agency's returned reference.

PENDING​

Please note that "success" on "resultType" means the Find API request was handled successfully and it doesn't mean the payment transaction result itself is "success." The status of the transaction itself is "pending", not "success."
