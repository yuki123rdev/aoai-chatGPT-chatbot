Overview​

This page is an overview of the services provided by PayVault team.

Introduction​

PayVault provides easy to implement tokenization solutions to Rakuten services.

The one is PayVault which is designed to tokenize card information.

The other one is InfoVault which is designed to tokenize sensitive file and text information.

In other words, PayVault caters to services that wish to store credit card and sensitive file and text information in a SECURE and PCI DSS compliant digital environment. In addition to PCI DSS scope reduction, PayVault supports legacy payment card data and allows easy customer identification by retaining the first six and last four card number digits.

How to Apply​

How to Apply - Service ID, API, payvault.js, CIT, etc.​

  1. Please read PayVault Introduction and Technical Guidelines.
  2. Please contact PayVault/Payment Platform team to discuss your needs and design.
  3. Please apply for Service ID if you have decided to use PayVault products with below form.
    Application for Product Use
  4. Please comment on same Application ticket about your request or reopen ticket if you want to request for additional PayVault products usage.
    • Request for additional PayVault API, infoVault API.
    • Request for CIT usage.
    • Request for ID Token Grouping status.
    • Request for Issuer code and card Type usage for SPU, etc.
    • Other Request.
  5. Please contact Rakuten ID team to check config on ID setting with your Service ID.

Implementation Process​

  1. Please read Guidelines and use PayVault API Testing Portal to get sample code.
  2. Create Grand Design/Specifications to share/check with Rakuten Security Team.
  3. After Security Team approval, Implement at your DEV/STG Environment.
  4. Schedule STG, QA Test and Data Migration if necessary for your service. https://payvault-tool-testing-dev.ap-northeast-1.elasticbeanstalk.com/testing

Technical Guideline​

PayVault APIs​

PayVault provides a set of APIs that can be called by your service in order to perform create, modify, or inquiry operations in the PayVault. These APIs are:

API NameDescription
/api/pv/Card/V1/AddGenerating card tokens by submitting card information.
/api/pv/Card/V1/AddPlusGenerating card tokens by submitting card information and return card fingerprint in the response.
/api/pv/Card/V1/UpdateUpdating card expiry date.
/api/pv/Card/V1/DeleteDeleting previously registered cards.
/api/pv/Card/V1/GetRetrieving non-sensitive card information given a card token.
/api/pv/Card/V1/RedeemRetrieving unmasked card number given a card token.
/api/pv/Card/V2/RedeemRetrieving unmasked card number and CVV given a card token and CVV token.
/api/pv/Fingerprint/V1/GetRetrieving fingerprint of card.
/api/pv/Card/V1/ForwardReplacing card token of request data from services with raw card number then sending to external service like Accertify.
/api/pv/Card/V2/AddGenerating card tokens by submitting card information. And issuerCode and cardType are returned in the response for allowed serviceIds.
/api/pv/Card/V3/AddSame as V2, but also generating CVV token by submitting CVV, together or without credit card number.
/api/pv/Card/V2/AddPlusGenerating card tokens by submitting card information and return card fingerprint in the response. And issuerCode and cardType are returned in the response for allowed serviceIds.
/api/pv/Card/V2/GetRetrieving non-sensitive card information given a card token. And issuerCode and cardType are returned in the response for allowed serviceIds.

InfoVault APIs​

InfoVault provides a set of APIs that can be called by your service in order to create, modify, or inquiry operations in the InfoVault. These APIs are:

API NameDescription
/api/pv/File/V1/AddGenerating file token by uploading a file.
/api/pv/File/V1/RedeemDownloading file with file token.
/api/pv/File/V1/GetRetrieving file information with file token.
/api/pv/Text/V1/AddGenerating text tokens by submitting sensitive information.
/api/pv/Text/V1/RedeemRetrieving text information with text token.

PayVault and InfoVault JavaScript Libraries​

The PayVault provides two JavaScript libraries. These libraries are:

Library NameDescription
/static/payvault/V4/payvault.jsCapturing a credit card information and/or CVV directly from the shopper's browser. The credit card data is submitted to the blank iFrame that is held by PayVault then the iFrame will submit credit card data by using AJAX to /api/pv/Card/V1/Add. The card token can be used in lieu of credit card numbers throughout all supported Payment Gateway APIs.
SAQ-A
By using you use V4 payvault.js on your website, your service needs to be SAQ-A which you need to meet about 22 PCI DSS requirements. See this link for more information.
/static/infovault/V1/infovault.jsCapturing file from the shopper's browser. The file data is submitted using AJAX to InfoVault Add File API. A card token will be returned and appended to the file form.

Announcements and Contacts​

To keep up to date with PayVault related news or to get in touch with the PayVault team please follow the instructions below.

Announcements​

PayVault related news will be released through the following mailing lists:

  • pro-payvault-service@mail.rakuten.com: PayVault regular announcements (e.g. regular maintenance, PayVault release schedule, IF change)
  • pro-payvault-service-E@mail.rakuten.com: PayVault critical announcements (e.g. service disruption) Please use the new DLM to join these mailing lists.
Please subscribe to both mailing lists if you are currently using the PayVault service.
