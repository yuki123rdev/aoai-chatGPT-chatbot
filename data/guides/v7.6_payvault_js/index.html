Installation​

Include the payvault.js version in your checkout page HTML.

If you're using payvault.js version less than 7, e.g V3 or V4, please refer to this guideline as start. Or contact PayVault team directly from this link

New features​

Add AutoComplete Attribute in iframes : Card Number, Expired Month, Expired Year​

PayVault.JS V7.6 has officially added support for AutoComplete, enabling users to automatically fill in their credit card numbers by either scanning them with the built-in camera feature on their mobile devices or selecting from saved cards in their device or browser.

To know how Payment Step (a product of Payment Platform) applies it, you can refer to the [Specification] PayStep - AutoFill Payment Card Information with Smartphone Camera.

Note: AutoComplete for Card Holder Name​

PayVault.JS does not provide any component for Card Holder Name.

To include AutoComplete for Card Holder Name, please add the following attribute in the HTML code:

<div>
  <label for="card-holder-name-element">Card Holder Name</label>
  <input name="cardHolderName" id="card-holder-name-element" autocomplete="cc-name" />
</div>

The key point is to add the autocomplete="cc-name" attribute in the Card Holder Name input field.

Update Card Brand Logo​

This feature behavior is the same as payvault.card.getBrandCode method, If you have set brandCodes which doesn't allow your customer to use, then the card icon will show the default one. For more information about the card band rule, please check here. Card Number Validation Rule

Option to enable/disable the display of card brand logos and cvv icon​

By adding logo parameter in cardNumber and cvv when you call the setup method, you can control showing icon or not. (default value is true)

disable card icon
const inputElementStyles = {
  base: {
    color: 'black',
    border: '1px solid black',
    'border-radius': '4px',
    'font-size': '22px',
  },
  valid: {
    color: 'green',
    border: '2px solid green',
    'border-radius': '4px',
    'font-size': '20px',
  },
  invalid: {
    color: 'red',
    border: '1px solid red',
    'border-radius': '4px',
    'font-size': '18px',
  },
  focus: {
    border: '1px solid violet',
    'border-radius': '4px',
    'font-size': '20px',
  },
};
const myElements = {
  cardNumber: {
    mount: '#my-card-mount',
    styles: inputElementStyles,
    logo: false
  },
  cvv: {
    mount: '#my-cvv-mount',
    styles: inputElementStyles,
    logo: false
  },
  expirationMonth: {
    mount: '#my-expiration-month-mount',
    styles: inputElementStyles,
  },
  expirationYear: {
    mount: '#my-expiration-year-mount',
    styles: inputElementStyles,
    format: 'YYYY'
  },
};
payvault.setup(myElements);

Troubleshooting​

What client might face and what they can do.

IssueTo do
1The expiration year field does not match the format on iOS safari.
e.g. 05/2026 → 20/2026

For YYYY year format: add placeholder for expirationMonth and expirationYear

placeholder
  expirationMonth: {
    mount: ...
    styles: ...
    placeholder: 'MM'
  },
  expirationYear: {
    mount: ...
    styles: ...
    placeholder: 'YYYY'
  }

For YY year format: add placeholder for expirationMonth and expirationYear

placeholder
  expirationMonth: {
    mount: ...
    styles: ...
    placeholder: 'MM'
  },
  expirationYear: {
    mount: ...
    styles: ...
    format: 'YY',
    placeholder: 'YY'
  }
2

In Safari browser, the auto-fill feature does not fill out all the fields in one time.

After clicking the built-in "Scan Card" button above the keyboard and scanning the credit card, only the Credit Card Number field is filled-in with data.

To auto-fill the expiration year and expiration month, the end-user has to select the input fields one by one.

This is an expected behavior for cross-iframes in Safari.

It will be improved in succeeding versions of PayVault JS.

If none of them solve your issue, please contact us using this form.
