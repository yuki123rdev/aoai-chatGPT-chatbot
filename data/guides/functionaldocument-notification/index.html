Notification​

We notify a payment result to you later if you set a URL at your side in notificationUrl in the following APIs.

/Payment/V1/Authorize

/Payment/V1/Authorize/HTML

/Payment/V1/AuthorizeAndCapture

/Payment/V1/AuthorizeAndCapture/HTML

The notification is sent with two parameters, paymentinfo and signature, as same as the APIs.

It happens only when status is updated successfully and doesn't happen when failure and pending.

The detail is as follows.

#APIPattern
1/Payment/V1/Authorize (*1)initialized　→　authorized
2/Payment/V2/Authorize/HTML (*1)initialized　→　authorized
3/Payment/V1/AuthorizeAndCapture (*1)nitialized　→　captured
4/Payment/V2/AuthorizeAndCapture/HTML (*1)nitialized　→　captured
5/Payment/V1/Modify (*2)authorized　→　authorized
captured　→　captured
6/Payment/V1/Captureauthorized　→　captured
7/Payment/V1/CancelOrRefundauthorized　→　initialized
captured　→　initialized

(*1) Happen even if grossAmount is 0.

(*2) Happen even if there is no change on payment status.

The notification is repeated until you respond HTTP status 200 and Http response as { "resultType":"suceess" } with a certain interval as explained below.

You can define how many times you'd like us to retry and how long intervals in advance.

We still recommend you to implement Find API because if we cannot get a response from you within the period you defined due to any reason, for example a temporary network trouble, you have no way to know if the transaction is updated.

Retry​

We'll notify the URL immediately once we get a trigger updating payment status.
We expect your system to respond HTTP status 200 and Http response as { "resultType":"suceess" } in that case.
If not, we'll retry up to 20 times with the following cycle.

The interval expands by 5 minutes.
If we notify 3 times, the 3rd one happens 15(=5*3) minutes later since the 2nd one.
The last 20th notification is notified 1050 minutes(17.5 hours) later since the 1st one.
