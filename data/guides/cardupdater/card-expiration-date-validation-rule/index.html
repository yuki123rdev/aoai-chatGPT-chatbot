Overview​

Card Updater service is used to update card information registered previously with card token.

File Server​

EnvironmentConnectionProtocolHostBucketPath
RequestStagingPublic InternetHTTPSs3-ap-northeast-1.amazonaws.comrpsp-payvault-cardupdater-staging{yourServiceId}/request/card-update/v1/
ProductionPublic InternetHTTPSs3-ap-northeast-1.amazonaws.comrpsp-payvault-cardupdater-production{yourServiceId}/request/card-update/v1/
ResponseStagingPublic InternetHTTPSs3-ap-northeast-1.amazonaws.comrpsp-payvault-cardupdater-staging{yourServiceId}/response/card-update/v1/
ProductionPublic InternetHTTPSs3-ap-northeast-1.amazonaws.comrpsp-payvault-cardupdater-production{yourServiceId}/response/card-update/v1/

*Amazon S3 access control list (https://docs.aws.amazon.com/AmazonS3/latest/dev/acl-overview.html)
*Please make sure that service needs to allow the access to rpsp-payvault-cardupdater-staging.s3.ap-northeast-1.amazonaws.com

File Format Requirement​

Data FormatLine Break TypeCharacter EncodingFile Name
RequestJSONLFUTF-8<any string or number>_<timestamp (yyyymmdd)>.json *
filename length should not exceed 40, excluding .json extension
ResponseJSONLFUTF-8res_<any string or number>_<timestamp (yyyymmdd)>.json **

danger

*Please note if a filename was already used before, the request will go in error to avoid multiple requests by mistake


　 <Request file>

  • As for the date (yyyyMMdd) in request file, we recommend setting the date on which the file is created or uploaded to S3 while you can set your preferred date.
  • Please avoid to set past date (e.g. 20000101) or future date (e.g. 21000101) since the operation may NOT normally proceed.

<Response file>

  • A timestamp of Response is the same as that of Request, regardless of the upload or pass date. (e.g. Request; {yourServiceId}_20221213.json Response: res_{yourServiceId}_20221213.json)
note

** Same as request with 'res_' as prefix

Specification of Request file​

IF Specification​

note

One Json Object per line

Field NameTypeLengthRequiredDescription
cardTokenstring32RA card token representing the given card number.
expirationMonthstring2R*The given card number's expiration month.

expirationMonth AND expirationYear must be set together. If you set only expirationMonth, missing expirationYear, you shall get the failure result in the given record.

*If you've already implemented/used Card Updater (before Sep 2022), it's not required (Optional).
expirationYearstring4R*The given card number's expiration year.

expirationMonth AND expirationYear must be set together. If you set only expirationYear, missing expirationMonth, you shall get the failure result in the given record.

*If you've already implemented/used Card Updater (before Sep 2022), it's not required (Optional).

R: Required, O: Optional

Max size

Allowed maximum 9.000.000 lines

Tokens authorization check

The tokens in this document are belongs to test service. Please use the tokens which belongs to your own service (or group). Otherwise you'll get unauthorized_access error.

Validation error

If data validation fails or JSON string is unreadable even for a single line, the batch will stop and won't send any data to JCN

Duplicates

In case of duplicated tokens, only one will be sent to JCN, the others will get "invalid_request_parameter" error: Duplicate card token in the Request file.


This error occurs as long as there's same tokens in the file. Please eliminate same tokens to avoid this error if possible.

Example Request File​

{"cardToken": "18121106001VftmBTz76ti1KmFVf9875", "expirationMonth": "12", "expirationYear": "2018"}
{"cardToken": "18121106001sG2ueZk1cDT8T4Gjk3460", "expirationMonth": "01", "expirationYear": "2019"}
{"cardToken": "18121106001th94C9tjDHErN3gN98561", "expirationMonth": "10", "expirationYear": "2020"}
{"cardToken": "18121106001eCYKGP8m0tDUoxJCx5565", "expirationMonth": "09", "expirationYear": "2021"}

Specification of Response file​

IF Specification​

note

One JSON Object per line

Field NameTypeLengthRequiredDescription
resultTypestring7YThe result of the request: "success" or "failure".
isCardUpdatedstring4~5NIf the card number was updated: "true" or "false".
isExpiryDateUpdatedstring4~5NIf the expiry date was updated: "true" or "false".
cardTokenstring32NThe card token in request file
cardCompanystring40NThe cardIssuerCode of the card company the card number was managed
maskedCardDetailsMaskedCardDetailsObjectNThis is always set when isCardUpdated or isExpiryDateUpdated is "true"
errorCodestring1~100NError code when resultType is "failure".
errorMessagestring1~100NError message when resultType is "failure".

MaskedCardDetails Object​

Field NameTypeLengthRequiredDescription
cardTokenstring32NA card token representing the given card number.
iinstring6NThe first 6 digits of the given card number (the Issuer Identification Number, also known as BIN or Bank Identification Number).
last4digitsstring4NThe last 4 digits of the given card number.
expirationMonthstring2NThe month of the expiration date, as given in the request.
expirationYearstring4NThe year of the expiration date, as given in the request.
brandCodestringNThe card brand. The brand is determined from the card number according to this table.
fingerprintstring37NThe card fingerprint.

Error CodeError messageNotes

not_foundThe given card token is not found.

card_blockedThe card was stopped by the cardholder.
