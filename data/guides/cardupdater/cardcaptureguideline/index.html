Overview​

Card Capture service is used to perform a card capture by a file.

File Server​

EnvironmentConnectionProtocolHostBucketPath
RequestStagingPublic InternetHTTPSs3-ap-northeast-1.amazonaws.comrpsp-payvault-cardupdater-staging{yourServiceId}/request/card-capture/v1/
ProductionPublic InternetHTTPSs3-ap-northeast-1.amazonaws.comrpsp-payvault-cardupdater-production{yourServiceId}/request/card-capture/v1/
ResponseStagingPublic InternetHTTPSs3-ap-northeast-1.amazonaws.comrpsp-payvault-cardupdater-staging{yourServiceId}/response/card-capture/v1/
ProductionPublic InternetHTTPSs3-ap-northeast-1.amazonaws.comrpsp-payvault-cardupdater-production{yourServiceId}/response/card-capture/v1/

File Format Requirement​

Data FormatLine Break TypeCharacter EncodingFile Name
RequestJSONLFUTF-8{yourServiceId}_<timestamp (yyyymmdd)>.json *
filename length should not exceed 40, excluding .json extension
ResponseJSONLFUTF-8res_<yourServiceId>_<timestamp (yyyymmdd)>.json **

*Please note if a filename was already used before, the request will go in error to avoid multiple requests by mistake
*If a client put multiple files with the same name, it will be only 1 file at the end, since newer files will override older files (same name).

<Request file>

  • As for the date (yyyyMMdd) in request file, we recommend setting the date on which the file is created or uploaded to S3 while you can set your preferred date.
  • Please avoid to set past date (e.g. 20000101) or future date (e.g. 21000101) since the operation may NOT normally proceed.
  • Please also check this note yyyyMMdd


<Request file>

  • A timestamp of Response is the same as that of Request, regardless of the upload or pass date. (e.g. Request; yourServiceId20221213.json Response: resyourServiceId_20221213.json)

Specification Request File​

IF Specification​

One Json Object per line

Field nameField name (Japanese)TypeLengthRequiredCheck Field by PayVaultValuesDescription
transactionCode取引コードstring-Y○
  • sale
  • cancel
The code of the transaction: "sale" or "cancel".
In case if you put neither of "sale" or "cancel", PayVault will print warn log and skip this record. Later you will get notification by email with failures records details.
transactionCategory取引区分string-Y○
  • single
Only single payment is accepted
subServiceIdstringup to 100Y - only if your service configured to use multiple merchants.
otherwise, no		○valid subserviceId providedif you're not sure about this point, please contact our team.
saleDate売上年月日string8Y○The date of the sale(capture) will be performed.
The format is "YYYYMMDD".
In case if you put wrong format, PayVault will print warn log and skip this record. Later you will get notification by email with failures records details.
saleAmount売上金額string7Y○The amount of sale for product.
paymentIdstringup to 29N○Alphanumeric and - _ .This field is optional, and will not included in response json file.
But will include in JCN request file.
cardToken-string32Y○-A card token representing the given card number.

△ means PayVault will only check when you set this field.

Allowed maximum 9.000.000 lines

Validation error

If data validation fails or JSON string is unreadable even for a single line, the batch will stop and won't send any data to JCN

Example Request File​

Without paymentId

{"cardToken":"17033002001GNkMpUjK8ZggW1kFm0434","transactionCategory": "single","transactionCode": "sale","saleDate": "20171204", "saleAmount": "01230"}
{"cardToken":"17033002001ohfYxiRf7476yjRhY3042","transactionCategory": "single","transactionCode": "cancel","saleDate": "20171204", "saleAmount": "01230"}

With paymentId

{"cardToken":"17033002001GNkMpUjK8ZggW1kFm0434","transactionCategory": "single","transactionCode": "sale","saleDate": "20171204", "saleAmount": "01230", "paymentId":"RCF-A-20211006-0703528625"}
{"cardToken":"17033002001ohfYxiRf7476yjRhY3042","transactionCategory": "single","transactionCode": "cancel","saleDate": "20171204", "saleAmount": "01230"}

Specification of Response file​

Response only contain the information on which token was sent to which card company

One Json Object per line

IF Specification​

Field nameTypeLengthRequiredDescription
resultTypestringYThe result of the request: "success" or "failure".
cardTokenstring32NThe card token in request file
cardCompanystringNThe cardIssuerCode of the card company the card number was managed
errorCodestringNError code when resultType is "failure".
errorMessagestringNError message when resultType is "failure".

Error code list​

Error codeError messageNotes
invalid_request_parameterCard brand is not supportedwon't throw this if a default card company is set
invalid_request_parameterValidation error: Card number longer than 16JCN doesn't support card numbers longer than 16 digits
invalid_request_parameterValidation error: Invalid paymentIdProvided paymentId is longer than 29 chars or doesn't start with R
not_foundThe given card token is not found.
unauthorized_accessAuthorization failedIf you use a token created by a service that you are not authorized to use

// Failed at PayVault
{"resultType": "failure", "cardToken": "9xy8Q6rDkVmZajkQ57a3KDGTBj18rU68", "errorCode" : "not_found", "errorMessage" : "The given card token is not found."}
{"resultType": "failure", "cardToken": "9xy8Q6rDkVmZajkQ57a3KDGTBj18rU69", "errorCode": "invalid_request_parameter", "errorMessage":"Card brand is not supported"}
 
// Success case
{"resultType": "success", "cardToken": "9xy8Q6rDkVmZajkQ57a3KDGTBj18rU68", "cardCompany": "2a99660"}
{"resultType": "success", "cardToken": "8d61CD5D4l5QYsGyA7z8lDgn7asHEY12", "cardCompany": "2a99661"}

Critical error list​

In case of these critical errors, a response file won't be generated, the batch is considered as failed and we won't send request file to JCN.

ErrorNotes
JSON string is invalidwrong format of the request file (not a json format...)
Transaction category is wrong
Transaction code is wrong
Transaction category is not single
File has invalid token (not 32 characters in length)
File is too big (> 9M lines in total)
Other validation errorsWrong field length, unexpected value..
