What is payvault.js​

PayVault provides easy to implement tokenization solutions to Rakuten services.

PayVault caters to services that wish to store credit card and/or sensitive file information in a SECURE and PCI DSS compliant digital environment. In addition to PCI DSS scope reduction, PayVault supports legacy payment card data and allows easy customer identification by retaining the first six and last four card number digits.

An additional tool, called InfoVault is used to check the sensitive file you added previously with the file token or requestId. Switch to the tool with the button on the upper right corner.

Signature check

It is mandatory to check signature sent in the response before saving or using the returned card token. If this check is missing, the response can be tampered creating a critical security issue.
More information on how to check the signature can be found here: PayVault and InfoVault API Implementation Guideline#Howtocheckresponsesignature

Guides​

Setting Up​

Step 1: Install
Ensure payvault.js has been added to the page (see Installation section for instructions)

Step 2: Setup DOM
Add mount points to checkout page html.

<h1>My Checkout Form</h1>
<div id="my-card-mount"></div>
<div id="my-cvv-mount"></div>
<div id="my-expiration-month-mount"></div>
<div id="my-expiration-year-mount"></div>

Step 3: Specify Options
Create options object with element options. (For more a complete list of available options please check the Setup Options section of the API Reference.)

const myElements = {
  cardNumber: {
    mount: '#my-card-mount'
  },
  cvv: {
    mount: '#my-cvv-mount'
  },
  expirationMonth: {
    mount: '#my-expiration-month-mount'
  },
  expirationYear: {
    mount: '#my-expiration-year-mount'
  }
};

Step 4: Setup payvault
Pass your options to the payvault.setup function.

payvault.setup(myElements);

Step 5: Confirm
If the setup is successful, you should see your payvault inputs rendered to their specified mount points . (See the Styling Inputs section for instructions on how to apply your custom styles)

Step 6: Check payvault installation
if payvault.setup() is called more than once use payvault.destroy()

payvault.destroy​

Remove all input iframes and coordination iframe

payvault.destroy();

Get a example for payvault.destroy method with react here

Creating Tokens​

Tokens are created by using the createToken() function. For more a complete list of available options please check the Create Token Options section of the API Reference. createToken() can be used with both with callbacks or promises.

Callback

payvault.card.createToken({ serviceId: 'YOUR_SERVICE_ID' }, function (token){
  console.log(token);
});

Promises

payvault.card.createToken({ serviceId: 'YOUR_SERVICE_ID' })
  .then(function(token) {
    console.log(token);
  });

Async or await

async function someFunction() {
  const token = await payvault.card.createToken({ serviceId: 'YOUR_SERVICE_ID' });
  console.log(token);
}

  },

note

Only the options above can be specified in the payvault options. All other values will be ignored by Payvault.

Event Listeners​

In the case you need to know the current state of your payvault inputs while users interact with them, payvault.js allows you to add eventListeners to inputs which are triggered on the chosen event, and return the current input state and meta data for you to do as you please. See event listeners section of API reference for full details.

payvault.card.addEventListener("cardNumber", "input", function(event) {
  if (!event.valid) {
    const errorContainer = document.getElementById('card-error-container');
    errorContainer.innerHTML = 'Oh no, that card is invalid';
  } else {
    errorContainer.innerHTML = '';
  }
});

    Styling​

    Internal Styling​

    Custom styles can be applied to payvault inputs by specifying styles in the setup options. Only the following whitelisted css styles will be applied.All others will be ignored.

    color
    border
    border-radius
    padding
    font-size
    font-family
    font-weight
    background

    note

    Due to browser inconsistencies, styling select inputs (expiration dates) is not officially supported. We cannot guarantee which stylings will be applied or how they will look across browsers. As such, we recommend using text inputs for expiration dates if you require custom styles.

    External Styling​

    payvault.js iframes are set to transparent, so you can style behind the iframes using vanilla CSS, giving the appearance that the iframes themselves are being styled.

    Style Statuses​

    {
      color: inherit;
      border: transparent;
      border-radius: inherit;
    }

    base styles

    Description: If base styles are set, these become the initial styles after setting up payvault.js.
    Fallback: css properties not specified, fall back to using code styles.

    valid styles

    Description: Styles configured in valid are applied, if the card element is in a valid state, and blurred.
    Fallback: css properties not specified, fall back to using base styles.

    Invalid styles

    Description: Styles configured in invalid are applied, if the card element is in an invalid state, and blurred.
    Fallback: css properties not specified, fall back to using base styles.

    focus styles

    Description: Styles configured in focus are applied, if the card element is in a focus state.
    Fallback: css properties not specified, fall back to using base styles.

    Bubbling up styles

    Styles will bubble up in this order: base -> focus -> valid | invalid

    payvault.setup​

    Sets up all the payvault.js apparatus, then the user can run any of the functions payvault.js provides for elements not provided, it is assumed the user will provide via iframe

    type Element = 'cardNumber' | 'expirationMonth' | 'expirationYear' | 'cvv';

    type DefaultBrandCodes = 'Visa' | 'MasterCard' | 'Diners Club' | 'Discover' | 'JCB' | 'American Express' | 'Maestro';
    type OptionalBrandCodes = 'Edy'

      cvv: ElementOptions

    payvault.card.createToken​

    payvault.card.createToken({
      attachResponseTarget: document.getElementById('my-form'),
      integrationCase: 'cardToken',
      maxRetrialTimes: 10,
      ... /* See options below */

    Signature check

    It is mandatory to check signature sent in the response before saving or using the returned card token. If this check is missing, the response can be tampered creating a critical security issue. More information on how to check the signature can be found here: PayVault and InfoVault API Implementation Guideline#Howtocheckresponsesignature

    Options​

    Any of these options can be passed to the payvault.card.createToken function. (Please see here for a list of options for payvault.setup)

    integrationCaseyescardTokenOptions: cardToken, cardTokenWithCvv, cvvToken
    maxRetrialTimesyes3Set maximum retrial times for payVault.card.createToken's Add Card API HTTP request (limit: 10)
    retryInterval (ms)yes100Set time interval between retries for HTTP requests to the Add Card API under payVault.card.createToken

    Helper methods​

    payvault.card.addEventListener​

    Add event listener to any of the card elements to inspect the current state of the input.

    NOTE (i):

    While "mousedown" | "mouseup" | "mouseenter" | "mouseout" events are supported by payvault v7, they may respond differently in mobile broswers. Where possible, we recommend using "blur" | "focus" | "input" events.

    NOTE (ii):

    Click events do not trigger on select elements in Safari.

    NOTE (iii):

    input event is an alias for change event. As such, you can apply input events to select elements.

    type ElementType = "cardNumber" | "cvv" | "expirationMonth" | "expirationYear";
    type InputEventType = "blur" | "focus" | "input" | "click" | "mousedown" | "mouseup" | "mouseenter" | "mouseout";
    type ElementEvent = ElementStatus & CardMetadata;

    payvault.card.addEventListener(ElementType, InputEventType, Function(ElementEvent));
    Event Values:​
    type BrandCode = "Visa" | "MasterCard" | "Diners Club" | "Discover" | "JCB" | "American Express" | "Maestro" | "Edy" | "Invalid" | "Unknown";

    // CardNumber Event
    {
      valid: boolean,
      length: string,
      brandCode: BrandCode
    }

    // CVV Event
    {
      valid: boolean,
      length: string,
      validCvvForCardNumber: boolean,
    }

    // Expiry Year Event
    {
      valid: boolean,
      length: string,
      validDate: boolean
    }

    // Expiry Month
    {
      valid: boolean,
      length: string,
      validDate: boolean
    }
    payvault.card.getBrandCode​

    Returns card brand code. However, if the card brand is not supported and the format of the card number is correct (only containing digits), then the returned value will be "Unknown". If the format of the card number is not correct, then the returned value will be "Invalid".

    // async
    const brandCode = await payvault.card.getBrandCode(); // "American Express"
    // callback
    payvault.card.getBrandCode(function(brandCode) { console.log(brandCode) });
    payvault.card.validateCardNumber​

    For more information about the supported card brand list and other details, please refer to Card Number Validation Rule.

    // async
    const isValid = await payvault.card.validateCardNumber(); // true | false
    // callback
    payvault.card.getBrandCode(function(isValid) { console.log(isValid) });
    payvault.card.validateCvv​

    Returns true for lengths 0-4

    // async
    const isValid = await payvault.card.validateCvv(); // true | false
    // callback
    payvault.card.validateCvv(function(isValid) { console.log(isValid) });
    payvault.card.validateCvvForCardNumber​

    Validates cvv based on the inputted card brand. If brand is unknown, will validate length only.

    // async
    const isValid = await payvault.card.validateCvvForCardNumber(); // true | false
    // callback
    payvault.card.validateCvvForCardNumber(function(isValid) { console.log(isValid) });

    Example:

    payvault.card.validateCvvForCardNumber(); // for cvv: 7198, cardNumber: "346956358241301" result: true
    payvault.card.validateCvvForCardNumber(); // for cvv: "", cardNumber: "6761-9550-2231-0263" result: true
    payvault.card.validateCvvForCardNumber(); // for cvv: "341", cardNumber: "3579 2081 7340 3878" result: true
    payvault.card.validateCvvForCardNumber(); // for cvv: "834", cardNumber: "6011-6097-7962-0113" result: false
    payvault.card.validateExpiration​

    Validates month and year (year must be within 19 years from current date)

    // async
    const isValid = await payvault.card.validateExpiration(); // true | false
    // callback
    payvault.card.validateExpiration(function(isValid) { console.log(isValid) });

    Example:

    /* Exectuted on 2020-07-01 */
    payvault.card.validateExpiration(); // for expirationMonth: "09", expirationYear: "2017" result: false
    payvault.card.validateExpiration(); // for expirationMonth: "01", expirationYear: "2020" result: false
    payvault.card.validateExpiration(); // for expirationMonth: "09", expirationYear: "2021" result: true
    payvault.card.validateExpiration(); // for expirationMonth: "01", expirationYear: "2039" result: true
    payvault.card.validateExpiration(); // for expirationMonth: "01", expirationYear: "2040" result: false
    payvault.card.validateExpiration(); // for expirationMonth: "01", expirationYear: "2099" result: false
