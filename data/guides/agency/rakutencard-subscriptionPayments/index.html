A: Alpha, AN: Alphanumeric, ANS: Alphanumeric Symbol, NS: Numeric Symbol, N: Numeric, B: "true" or "false", T: Text

  1. It expresses which item is used by which service for a fraud check. All fields are in "Fraud" with a "P" to be sent to Payguard (Transaction Guard) if you send the "P" marked parameters of your payment to PGW.
  • For Payguard, you need to ask the PGW team in advance if your service wants to use Payguard or not. We set up your configuration to allow your transactions to be sent to PayGuard. Without this configuration with your ServiceID on the PGW, fraud checks by Payguard don't work even if you set related parameters to your payment request.

Sample Request​

Sample JSON request of Subscription Payment
{
    "serviceId":"payStep1",
    "subServiceId": "payStep1",
    "timestamp":"2016-12-02 04:40:44.000",
    "paymentId":"PAY_STEP1_20161202044044",
    "agencyCode": "rakutencard",
    "currencyCode":"JPY",
    "grossAmount":"200",
    "callbackUrl":"https://mydomain.com/callback",
    "notificationUrl":"https://mydomain.com/notification",
    "subscription":{
        "version":"1",
        "amount":"200",
        "subscriptionId":"sub_0001"
    },
   "order":{
        "version":"1",
        "email":"test@rakuten.com",
        "ipAddress":"123.123.123.123",
        "items":[
            {
                "id":"item1",
                "price":"200",
                "quantity":"1",
                "name":"testItem1"
            }
        ]
    }
}
