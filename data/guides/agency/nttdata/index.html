transStatusthreeDSecureAuthenticationStatusNote
Yfully_authenticated3DSecure successful
認証 / 口座確認に成功
Nauthentication_error3DSecure failure (due to issuer or cardholder)
未認証 / 口座未確認；取引拒否
Uconnection_error3DSecure failure (any other reason)
認証 / 口座確認を実行できなかった；AResまたはRReqで示される技術的な問題またはその他の問題
Aeligible_for_3d_secureAttempt (3DSecure authentication attempted successfully)
処理の試行が実施された；認証 / 確認は行われなかったが、認証 / 確認を試行したことの証拠は提供されている
Cnot_eligible_for_3d_secure3DSecure failure
チャレンジが必要；CReq/CResを利用した追加の認証が必要とされる
Rnot_eligible_for_3d_secure3DSecure failure (rejected by the issuer)
認証 / 口座確認が拒否された；イシュアーが認証 / 確認を拒否しており、承認を試行しないよう要求している
Dconnection_error (*1)3DS2.2 new
チャレンジが必要です。Decoupled認証が確認されました。
Iconnection_error (*1)3DS2.2 new
情報提供のみ。 3DSリクエスターからのチャレンジ要求が承認されました
(*1) As of Jun. 15th. 2023, both values for version 2.2 should not be returned since they are designed for unsupported features by us.
  • "D" may be returned for 3RI transactions which are not supported by NTTData's 3DS Connector currently
  • "I" may be returned for the requests that have challengeIndicatorType "05", "06" or "07" which are not supported by PGW currently.
