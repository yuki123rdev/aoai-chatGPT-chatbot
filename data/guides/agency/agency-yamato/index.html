Custom field​

Not supported, please be careful about the difference between Jaccs and Yamato.

ServiceReferenceID

Yamato's orderNo can be set from serviceReferenceId in your Authorize request.

  1. Must be a combination of half-width numbers (0~9), alphabetic characters (a~z, A~Z), "-" [hyphen] or "_" [underline].
    半角の数字(0~9)または英字(a~z,A~Z)または”-”[ハイフン]または”_”[アンダライン]の組み合わせであること。
  2. Must be within the specified number of digits (20 digits).
    指定桁数(20 桁)以内であること。
  3. Must be unique among all payments/transactions made by the merchant. (the merchant means that the contract unit basis with Yamato)
    加盟店の全決済において一意(ユニーク)であること。

OrderNo Re-use

Specification of OrderNo (受注番号)

It is reusable under these 2 conditions.

  • Canceled
  • Payment overdue FYI: (Shared from Yamato)

受注番号の再利用ですが、加盟店および購入者がとった行動によって、再利用の可否が変わります。テスト環境・本番環境同様になります。

 【与信を取消したために管理画面非表示】 

   → 受注番号を再利用できます。

 【入金しないまま期限切れになったために管理画面非表示】

   → 受注番号を再利用できます。

 【『入金済』になってから長期間経過したために加盟店管理画面非表示】

   →  受注番号を再利用できません。

     理由は、入金済になった与信情報は、取消できない仕様になっており、その期間の期限がないため。

【補足】

 受注番号の重複エラーは、「同じ加盟店コードで同じ受注番号を入力したとき」に発生します。

 別の加盟店コードで同じ受注番号を入力しても、重複エラーにはなりません。

Modify API​

Payment MethodDescription
PostPay
  • Both 'authorized' and 'captured' status transactions are available to modify their amounts (Not possible to update "deliverySlipNo", so please do it through Yamato's admin console if necessary)
    Explanation from Yamato請求金額変更 API Process Flow inside Yamato
      

    新しい決済（金額変更時の決済）が成功した場合は既存決済（初回決済時の与信）がキャンセルされ、
    金額変更が正常に終了しなかった場合は既存決済（初回決済時の与信）が残ります。


    Captured transactions can be modified under the specific status.

    Increase：Within the date of capture request, afterward errorCode kaakk011E

    Decrease：Before being status paid "入金済み", afterward errorCode kaakk011E


    In case of unmatched process issue for captured transactions

    Increase
    各入金チャネルから入金情報が連携され11:請求書発行済み⇒12:入金済にステータスを変更する際に、クロネコ代金後払いサービス側で保持している金額と連携された入金情報を突き合せる処理がございます。
    この突合処理で相違が発生しますと12:入金済にステータスの変更が行われず、金額変更後の金額で請求書が発行されます。

    Decrease
    元の与信額10000円⇒変更後金額5000円に金額変更し、元の与信額10000円で入金されてしまった場合の返金については下記になります。

    1. 10000円の入金分については弊社で確認作業を実施し、変更後金額5000円の与信に入金処理、余った5000円については当社から購入者様へ返金もしくはほか債権があれば充当します。
    2. 金額変更後に発行された請求書でもしも購入者様が再度入金してきてしまった場合、当社から購入者様へ返金等の対応を行います。
    ※なお、返金事務の際には振込手数料等の事務手数料として330円を差し引いての返金をしております。

  • API cannot be called 60 days after the last authorized date.
  • the order.items are bundled into a single item setting item name "ご購入商品一式" on Modify API if authorized order Items name was over 1.
    e.g.) item.A 100 and item.B 50 → Modify to amount=80 → item.price=80, item.name="ご購入商品一式"
  • If the request amount is as same as authorized amount or captured amount, the request is rejected and it gets an error "unacceptable_request".
