CancelOrRefund API​

Payment MethodDescription
Rakuten PointThe acceptable period for cancellation and refund of your transaction is depended on the main payment method the shopper used, such as credit card payment.
When you get an error with the following API when making a payment request but not completed yet, Rakuten Points are released soon so the shopper can retry with their Rakuten Points for their payments.
  • /Payment/V1/Authorize
  • /Payment/V1/Authorize/HTML
  • /Payment/V1/AuthorizeAndCapture
  • /Payment/V1/AuthorizeAndCapture/HTML
Granted Rakuten Points
  • You can decide when you grant Rakuten Point to the shopper with your application setting, which you submit to the Rakuten Point team.
  • If you call this CancelOrRefund API after the period of granting Rakuten Point, you cannot retrieve it back though the request itself finishes successfully.
Redeemed Rakuten Points
  • If the shopper uses the standard Rakuten Point, the redeemed Rakuten Points are returned to the shopper without any problem.
  • However, if time-limited Rakuten Points are used for redemption and the Points have expired when calling the CancelOrRefund API, the redeemed Rakuten Points will not be returned to the shopper because they have already expired. Then the request itself of calling the CancelOrRefund API finishes successfully.

Modify API​

Payment MethodDescription
Rakuten Point (rakutenpoint_jp)Up to 170 days after authorized If you also use a credit card for your transaction, a shorter one is taken priority.
Rakuten Point (rakutenpoint_jp)
There are some limitations.
  1. Once the Rakuten Points are granted to the shopper, you cannot modify the granted Rakuten Points. If you call Modify changing rakutenPoint.grant.points after the period, you get an error as unacceptable_request.
  2. When you reduce the amount from a transaction mixed with a main payment method (like a credit card) with Rakuten Point, we reduce it from the main payment method, not from the redeemed Rakuten Points. This is because if the Rakuten Point is a term-limited Point, which has an expiration, and it's expired, actually, it's not backed to the shopper. The main-payment-method-first-rule prevents the situation as much as possible.
