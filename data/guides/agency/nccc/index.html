Note for Non-3DS Transaction​

For a merchant who has enabled 3DS service, if NCCC did not respond SecureStatus = Y (success) (EX: N (failure) or B (didn't go through 3D authentication)), PGW will send a failure response to the merchant and also send a request to NCCC to auto cancel the transaction later.

PAYINQ-6466 - [Development Request][tw-gecp] NCCC: Auto-Cancel on Non 3DS Transactions IN PROGRESS

Capture API​

Capture Delay setting​

Card and UnionPay: 168 hours (=7 days)

Capture acceptable period​

Payment MethodDetails
Credit CardCapture is acceptable up to 20 days after authorization.
  • e.g. Transactions made during Sep-1 by Taiwan local time can be captured by the end of Sep-20 by Taiwan local time.
UnionPayCapture is acceptable up to 5 days after authorization.
  • e.g. Transactions made during Sep-1 by Taiwan local time can be captured by the end of Sep-5 by Taiwan local time.
Credit Card/UnionPayBackground file transfer on the production is not executed for merchants whose motherMerchantId is in the following list. These are for testing and we can ignore error related to the ids.
  • 0000000000/ when merchant need disable credit card, BU need input dummy value to ignore validation
  • 1234567890
  • 3333333333
  • 6666666666
  • 7777777777
  • 9999999999
  • 0100811376 / old merchantId for merchants, now active merchant not to use
  • 6600800020 / demo shop
ncccUnionPayCaptureDate and ncccCardCaptureDate cannot be gotten by the Find API accordingly.

CancelOrRefund API​

Payment MethodDetails
Credit CardCancel/Refund
  • 730 days at NCCC.
  • There is no limit on the PGW system side to call the CancelOrRefund API, but the shopper's card account still needs to exist for the card issuer to accept the refund request.
UnionPay
Alert
NCCC doesn't accept requests for cancellations and refunds from 23:00 to 24:00 every day (Taiwan local time). Hence, PGW responds temporarily_unavailable when you call the CancelOrRefund API during the time accordingly.
Cancel
  • 0:00 - 23:00 (CST) It is possible to proceed with authorization and calling the CancelOrRefund API.
  • Transactions made on the day can be cancelled until 23:00 on the same day, when NCCC starts its cut-off process at the time.
  • After NCCC starts the cut-off, they no longer accept cancellation requests.
  • 23:00 -24:00 (CST) : The CancelOrRefund API is not available during this period.
  • Due to their daily cut-off processing, transactions made from 23:00 to 24:00 cannot be cancelled.
  • After that time, it became possible to call the Caapture and then the CancelOrRefund API. If you want to cancel it, you need to call the Capture API first and then do a refund by calling the CancelOrRefund.
  • If you call the CancelOrRefund API without capture, PGW responds unacceptable_request.
Refund
  • If you want to refund the transaction, please do it within one year.
