CancelOrRefund API​

Payment MethodDescription
Credit CardRefund
  • If a refund request happens after the following settlement processing, a credit note (AKADEN in Japanese, 赤伝) will be taken place on the Rakuten Card side by following the cut-off date (Shimebi, 締め日)
    • VISA and Mastercard: Settlement data transferred: Three business days
    • Other card brands - JCB, Diners, AMEX: Twice a month
  • The period allowed for a refund is up to 170 days after being captured. You get unacceptable_request after the period has passed.
Apple PayThe period allowed for a refund is up to 170 days after being captured.
Convenience Store Payment
  • You can execute the CancelOrRefund API when the transaction status is "authorized". However, if the transaction status is already "captured", you will get an unacceptable_request response because the refund after captured is not supported.
  • You may get an error even when "authorized" because there is a time lag until we get a notification to update to "captured" after the shopper makes a payment at a convenience store though it is rare. If it happens, you will also get an unacceptable_request response. And you will need to issue a refund manually.

Result values for japanBillingAddressResult​

PGW maps the response result as the following table. For example, when RakutenCard returns "0", PGW returns as "match".

Note that JapanBillingAddressResult is not returned in the following cases.

1) When card authorization fails

2) When japanBillingAddress is not set in the request

3) When Rakuten Card does not return a success response.

firstNameKana and lastNameKana have dependencies.

If either case is a "mismatch", both will become "mismatch" because Rakuten Card checks them as a string boding them.
