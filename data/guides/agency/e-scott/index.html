Capture API​

Payment MethodDescription
Credit Carde-SCOTT allows capturing for up to 7 months after the authorization is succeeded. After the period expires, the authorized transaction is deleted on the e-SCOTT side.

CancelOrRefund API​

Payment MethodDescription
Credit CardCancel/Refund
  • It is acceptable for up to 7 months since the authorization was succeeded.
  • After the period, the authorized transactions will be deleted on the e-SCOTT system side.

Specification for japanBillingAddress (Cardholder's attribute verification)​

This optional field validates the user's information, such as the cardholder's name and last name, the last four telephone numbers, and so on, as shown below.
Notes:

japanBillingAddress is not available in the following two cases.

  1. 3D Secure
  2. Pending
       a. It's due to the specification of e-SCOTT. They don't respond to japanBillingAddress with their request API, which we internally call to clear the pending while it responds to the initial payment request. If a transaction results in "pending" for any reason, japanBillingAddress is not included in the response from our Find API to check the latest transaction status.
       b.It's due to the specification of e-SCOTT. They don't respond to japanBillingAddress with their request API, which we internally call to clear the pending while it responds to the initial payment request. If a transaction results in "pending" for any reason, japanBillingAddress is not included in the response from our Find API to check the latest transaction status.

Reference​

"reference":{

  }
The following parameters on API response are optional, and you need to ask PGW to enable them with reasonable reasons if needed on your side.
  1. hasMemberId (*) : true or false if the shopper has MemberID or not.
  2. memberId (*) Contact us if you need it. We will need to check if you have the privilege of getting it. The value is encrypted.
  3. hasCvvToken: true or false, if cvv is set with the associated payment transaction
