Overview​

There is a product you can use.

ProductRole
APIRemittance API enables you to have remittance function.
Before calling the API, you need to have a remittance amount and the destination.

Common Knowledge​

Agency Code​

Anency CodeDescriptionSupported Remittance Method
rakuten_bankBank Transfer from Rakuten Bankbank
rakuten_pointGranting Rakuten Cash from Rakuten Point
Rakuten Super Point isn't supported to comply with the law.		rakuten_cash

Remittance Status Type​

The following statuses express a transaction situation.

Remittance Status TypeDescription
initializedA request was denied or is being processed by Create API.
authorizedA request was approved by 'Create API', but the result isn't fixed yet.
The status will be changed automatically to 'initialized' or 'captured' when the result is fixed.
capturedThe transaction was completed successfully.

Remittance Method​

Remittance Method CodeDescriptionStatus Transition Diagram
bankBank Transfer from Rakuten Bank
You need to have your corporate account of Rakuten Bank and apply to use "Real-time Bank Transfer" service provided by Rakuten Bank.
A remittance is made from the bank account to shopper's bank account.
As a payee's bank account, it supports Rakuten Bank and other banks like Mizuho, UFJ.

danger
If you remit to an other bank's account not Rakuten Bank as payee, the unwinding of funds transfer (組戻) might be happened. Even if you receive "success" as result of the transaction, the result will be changed to "failure" after that. You need to receive your notification, or call periodically Find API to retrieve the latest result of the transaction. The result should be fixed within 2 weeks.
danger
Rakuten Bank checks a transmission source IP address, you have to set our global IP address to your corporate bank account to enable us to connect Rakuten Bank in advance.
The following status transition depends on payee's bank account.
rakuten_cashRakuten Cash

You need to have your account of Rakuten Cash.
A remittance is made from the account to shopper's Rakuten account.

Request Type​

Request TypeDescription
createWhen Create API is called
captureWhen authorized status is changed to initialized or captured

Result Type​

Result TypeDescription
successA transaction was successful.
failureA transaction failed due to some reasons.
You can find the reason by the following error code.
pendingResponds when a tentative network error happened.
Basically it'll be updated to success or failure by our system.

Notification Result Type​

Result TypeDescription
successA notification was successful.
pendingA notification is preparing.
expiredA notification failed due to some reasons despite many retries.

Error Code​

Error Codesituation of ErrorYour Action after receiving Error Code
invalid_request_parameterRequest doesn't follow the validation rule.Check whether there is any mistake in the request.
invalid_remittance_methodIncorrect remittance method or the destination.Change the remittance method or the destination.
already_completedThe requested transaction has already been previously completed.Treat the request as processed and end the payment processing.
unauthorized_accessA fraud user tried to perform remittance.Do not process the remittance.
not_foundCannot find any matching item for the specified criteria.Check whether the remittance transaction record exists in your system.

If you have the transaction, please request that again.
temporarily_unavailableDue to system maintenance, your request is temporarily restricted.Send the request again later.
system_errorAn unexpected error has occurred in our server.Contact us to get the operation support.

Encryption Instructions​

Specification​

datavalue
Encryption schemeAES 256 bits
Block cipher mode of operationGCM (Galois/Counter Mode)
PaddingNo Padding
Encryption KeyWill be provided by Remittance Gateway. (hexadecimal representation)
IV (Initialization Vector)Should be generated each transaction. (12 bytes)
Representing raw byte dataBase64

How to encrypt the information used in your request​

1.Decode your "Encryption Key" (hexadecimal string) to the raw byte data. (32 bytes)
2.Generate an "IV" each transaction. (12 bytes)
3.Encrypt your data with 1 and 2.
4.Add the generated IV to the first encrypted raw byte data.
5.Encode the data by using Base64.

Sample Code (Pure Java on the version 8 and 11)
import java.util.Base64;
 
import javax.crypto.Cipher;
import javax.crypto.spec.SecretKeySpec;
 
public class RemittanceInfoEncoder {
 
    public static void main(String[] args) throws Exception {
 
        String plainText =
            "{ \"key1\" : \"value1\", \"key2\" : \"value2\" }";
 
        // 1. Decode your "Encryption Key" (hexadecimal string) to the raw byte data. (32 bytes)
        String encryptionKey = "00A8811C6A1A56F6B8824447E6F89EF4421894776AAF428C8CE8676117CA97E1";
        byte[] encryptionKeyBytes = fromHex(encryptionKey);
 
        // 2. Generate a "IV" each transaction. (12 bytes)
        Cipher cipher = Cipher.getInstance("AES/GCM/NoPadding");
        cipher.init(Cipher.ENCRYPT_MODE, new SecretKeySpec(encryptionKeyBytes, "AES"));
 
        // 3. Encrypt your data with 1 and 2.
        byte[] encryptedRawBytes = cipher.doFinal(plainText.getBytes("UTF-8"));
 
        // 4. Add the generated IV to the first encrypted raw byte data.
        byte[] iv = cipher.getIV();
        byte[] addedIvEncryptedRawBytes = new byte[iv.length + encryptedRawBytes.length];
 
        System.arraycopy(iv, 0, addedIvEncryptedRawBytes, 0, iv.length);
        System.arraycopy(encryptedRawBytes, 0, addedIvEncryptedRawBytes, iv.length, encryptedRawBytes.length);
 
        // 5. Encode the data by using Base64.
        String encryptedText = Base64.getEncoder().encodeToString(addedIvEncryptedRawBytes);
 
        System.out.println(encryptedText);
 
    }
 
    /**
     * Converts hexadecimal strings to byte digits.
     *
     * The alternative way of this method is to use {@code DatatypeConverter.parseHexBinary(encryptionKey);}.
     * Java version 9 and above require JAXB module. Java version 8 and below have JAXB module as standard.
     *
     * @param hex
     *            hexadecimal strings
     * @return byte digits
     */
    private static byte[] fromHex(String hex) {
 
        if (hex == null) {
            return new byte[0];
        }
 
        byte[] bytes = new byte[hex.length() / 2];
        for (int index = 0; index < bytes.length; index++) {
            bytes[index] = (byte) Integer.parseInt(hex.substring(index * 2, (index + 1) * 2), 16);
        }
 
        return bytes;
 
    }
 
}

How to decrypt the information used in our response​

1.Decode the encrypted text by using Base64.
2.Get the "IV" from the first 12 bytes of the encrypted text.
3.Remove the "IV" from the encrypted text.
4.Decode your "Encryption Key" (hexadecimal string) to the raw byte data. (32 bytes)
5.Decrypt the data with 2, 3, and 4.

Sample Code (Pure Java on the version 8 and 11)
import java.util.Arrays;
import java.util.Base64;
 
import javax.crypto.Cipher;
import javax.crypto.spec.GCMParameterSpec;
import javax.crypto.spec.SecretKeySpec;
 
public class RemittanceResultDecoder {
 
    public static void main(String[] args) throws Exception {
 
        String encryptedText =
            "l3GQzmPSnkfS4yJ8owIfyfs+VfU8y4oYJViy901GbdnZyy8+nYCYTOWhhp2R+3gAbTSj7ozunuK5ukl/vaHzqCdrFq4=";
 
        // 1. Decode the encrypted text by using Base64.
        byte[] addedIvEncryptedRawBytes = Base64.getDecoder().decode(encryptedText);
 
        // 2. Get the "IV" from the first 12 bytes of the encrypted text.
        byte[] iv = Arrays.copyOfRange(addedIvEncryptedRawBytes, 0, 12);
 
        // 3. Remove the "IV" from the encrypted text.
        byte[] encryptedRawBytes =
            Arrays.copyOfRange(addedIvEncryptedRawBytes, iv.length, addedIvEncryptedRawBytes.length);
 
        // 4. Decode your "Encryption Key" (hexadecimal string) to the raw byte data. (32 bytes)
        String encryptionKey = "00A8811C6A1A56F6B8824447E6F89EF4421894776AAF428C8CE8676117CA97E1";
        byte[] encryptionKeyBytes = fromHex(encryptionKey);
 
        // 5. Decrypt our response with 2, 3, and 4.
        Cipher cipher = Cipher.getInstance("AES/GCM/NoPadding");
        cipher.init(Cipher.DECRYPT_MODE, new SecretKeySpec(encryptionKeyBytes, "AES"), new GCMParameterSpec(128, iv));
 
        byte[] plainTextRawBytes = cipher.doFinal(encryptedRawBytes);
        String plainText = new String(plainTextRawBytes, "UTF-8");
 
        System.out.println(plainText);
 
    }
 
    /**
     * Converts hexadecimal strings to byte digits.
     *
     * The alternative way of this method is to use {@code DatatypeConverter.parseHexBinary(encryptionKey);}.
     * Java version 9 and above require JAXB module. Java version 8 and below have JAXB module as standard.
     *
     * @param hex
     *            hexadecimal strings
     * @return byte digits
     */
    private static byte[] fromHex(String hex) {
 
        if (hex == null) {
            return new byte[0];
        }
 
        byte[] bytes = new byte[hex.length() / 2];
        for (int index = 0; index < bytes.length; index++) {
            bytes[index] = (byte) Integer.parseInt(hex.substring(index * 2, (index + 1) * 2), 16);
        }
 
        return bytes;
 
    }
 
}
