Payment Step Configuration

Payment Step creates "Service ID" when you apply for application.
You can customize the following configurations.

#ConfigurationDescriptionExamples
1Time-Zone SettingTime-Zone used by "date" and "datetime" objects such as transaction datetime are displayed on Payment Step screens and completion email (which is sent by Payment Step after the payment has completed).JST
2Supported LanguagesEnable the language switch on pages.
Below languages are supported by PayStep.

en: English
ja: Japanese
zh-tw: Traditional Chinese
zh-cn: Simplified Chinese
ko: Korean		ja,en,zh-cn,zh-tw,ko
3On/Off for Rakuten Member CheckoutTrue : Support Rakuten Member Checkout
False : Not support Rakuten Member Checkout

If you don't want Rakuten Member to pay via Payment Step, you can turn off the Rakuten Members Checkout function.

If you set to false, login page will be skipped when Shoppers enter to Payment Step.		true
4On/Off for Guest CheckoutTrue : Support Guest Checkout
False : Not support Guest Checkout

If you don't want Guest users to pay via Payment Step, you can turn off the Guest Checkout function.		true
5On/Off for Payment Completion EmailTrue : Send payment completion email
False : Not send payment completion email

If you don't want Shopper to receive the payment completion email from the Payment Step, you can turn off the function.		true
6BCC Setting for Payment Completion EmailAs internal BU staff, if you want to receive the payment completion email from the Payment Step as Bcc, you can specify the address.Rakuten Internal mailing list
7Customized Template of Payment Completion EmailIf you want to customize a payment completion email sentence as a email template, you can determine the email sentence for each language.
  • It supports text email only
  • You can embed the following parameters into mail template.

    user name
    payment ID
    transaction time (yyyy-MM-dd HH:mm:ss)
    gross amount
    used point amount
    main payment amount (= grossAmount - usedPointAmount)
    Item's ID, name, price, quantity

    Error will be show if the payment method is not available due to the minimum amount:
    • Gross amount is less than minimum amount
    • point payment amount is less than minimum amount
    • Minimum amount of credit card is mandatory

*1. The detailed information of preparing / issuing / using coupons​

Coupons

Steps to Support Coupon on PayStep

  1. Contact Coupon Team to get settings (System Id, Access Key)
    Coupon Platform

  2. Get Access to Coupon Management Tool (Suzaku) - This tool is managed by Rakuten Coupon Team
    For STG you need to setup PROXY: stg.checkproxy.rakuten-it.com / 9502)

  3. Create Coupon

  • a.Access Coupon Management Tool (Suzaku)

https://collabo.rakuten.co.jp/adfs/input

  • b.Create the Coupon
    Currently PayStep only supports service coupon, so please select couponType = "Service Coupon" and Discount Applicable target = "Service" check and input the mandatory field
    Click "SAVE", and then click "CREATE"

  • c.Click the "Confirm Create" to finish creating the coupon.

  • d.Set the MINIMUM amount (i.e 1200 JPY). In this example users can only use this coupon when PayStep grossAmount in the request parameters is JPY 1200 or more

  • e. Once you click the "Complete Coupon", the coupon will be issued immediately.

  1. Get the Coupon URL (URL Tab)
    Once the end-user access the "Acquisition" URL, the user will get the coupon and it will be available on PayStep after user logins on PayStep.
    Please note that the MINIMUM amount is used on PayStep to decide if the user can or not use this specific coupon (grossAmount less than JPY 1200 (i.e. JPY 1199) will not be eligible for this coupon and user will not see even if user clicked and accepted the coupon)
note

Notes for Coupon:

  • PayStep shows up to 10 coupons. (Users with over 10 eligibles coupons will see only the top 10 sorted by higher discount)
