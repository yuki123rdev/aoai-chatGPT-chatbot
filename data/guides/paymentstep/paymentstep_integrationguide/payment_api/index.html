Introduction​

It is the main API provided by Payment Step, it allows you to process a transaction with various payment method with a simple JSON request.
Simply speaking, Payment Step provide a simplified and visualized version of Global payment Authorize(Rakuten Card,RakutenPoint (japan),RakutenPoint (Global),Rakuten Coupon,PayPal) and AuthorizeAndCapture(Rakuten Card,RakutenPoint (japan),RakutenPoint (Global),Rakuten Coupon,PayPal).
Payment API support use to pay with follow:

Shopper identityPayment methodSub payment method
Rakuten MemberCredit CardRakuten Point
Guest UserPaypalRakuten Coupon
Alipay
Mobile carrier
Cash on Delivery

Working Example​

1.Generate a json request based in the Payment Step Payment API.
To make the integration smooth, you may start with the simplest form of request.

Payment Simplest Simple Request
 {
  "serviceId": "stg-all-paystep",           # set with your service id
  "subServiceId": "stg-all-paystep",        # same as service id if you did not request any special requirnment with Payment Step team
  "paymentId": "paymentId20171117145557034",# do forget to use a new id if previous payment was successed
  "currencyCode": "JPY",
  "grossAmount": 1000,
  "timestamp": "2017-11-17 14:55:57.034"    # set it within 5 minutes of system time
 }

2.Encode and encrypt the json request to base64 and SHA256
3.Return a redirect page to shopper with parameters calculated in step 2), and post to Payment Step Payment API endpoint

Redirect page example
# Redirect page example
<body onLoad="document.form.submit();">
    <form name="form" method="post" action="https://stg-step.payment.rakuten.co.jp/step/payment/V1">
        <input type="hidden" name="paymentInfo" value="jsonRequestInBase64"/>
        <input type="hidden" name="signature" value="jsonRequestWithSHA256"/>
    </form>
</body>

4.If Shopper can see the login page, it means your application is successfully connected to paystep.
So try to complete payment with different payment method with test account

5.Back to Step 1 but this time you should try with the full Payment API. Check which fields are required for your application.
