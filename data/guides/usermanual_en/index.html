Intended Use​

The Seller Portal is a tool to manage transactions created using APIs provided by us. The Seller Portal is mainly used for the following cases.

  • Obtain an authentication key provided by us
  • Change the amount, confirm sales, or cancel payment transactions
  • Check the results at the time of testing
  • Confirm details of the subject transaction when the user inquires

How to sign-in​

Rakuten staff​

For those who have Rakuten INTRA account

(1) Push "SIGN IN" button.

Outside Rakuten staff​

(1) Enter your e-mail address which your administrators granted the permission.
(2) Enter your password.
(3)Push "SIGN IN" button.
Then you will receive an email written in an one-time password.


(4) Copy the one-time password (6 digits) written in the e-mail you received.

(5) Paste the copied one-time password.
(6) Push "VERIFY" button.


If the verification failed 5 times, you return to Sign-in page. Then you can retry sign-in, but the one-time password will be changed.


How to logout​

(1) Place the cursor on the button on the right side of the header.

(2) Click the "Logout" button.


How to reset password​

If you are Rakuten staff, you cannot reset your password via this procedure (because your account is managed by Okta).


(1) Push "Forgot your Password" link.

(2) Enter your e-mail address.
(3) Push "SEND" button.
Then you will receive an e-mail written in an one-time password.


(4) Copy the one-time password (6 digits) written in the e-mail you received.

(5) Paste the copied one-time password.
(6) Push "VERIFY" button.


If the verification failed 5 times, you return to Sign-in page. Then you can retry password-reset, but the one-time password will be different.

(7) Enter a new your password.

  • Not allow blank spaces.
  • The minimum length is 8.
  • The maximum row is 128.
  • Allowed symbols are : ~ ! @ # $ % ^ & * ( ) _ + | } { [ ] \ > < ? : " ; ' , . / = -
  • Case sensitive.
  • It should be used at least three character types (uppercase letters, lowercase letters, numbers, symbols).
  • It should be different from the user's email address.

(8) Enter the new password the same as (7).
(9) Push "SAVE" button.


(10) Push "Return to sign in page" link.

How to check operation history​

You can authorize users to check their operation history in the "User List" of the "User Management" screen.
The operation history allows you to check the following

  • Change account settings
  • Password Change
  • Payment history retrieval
  • CSV file output
  • Changes in payment processing (amount changes, sales confirmation, cancellations/refunds)

(1) Place the cursor at the right edge of the header to display the submenu.
(2) Click "User Management" in the submenu to display the "User List" screen in the "User Management" section.

(3) Click the "Add User" button.

(4) Enter the email address of the user you wish to authorize.
(5) Check the "Operation Log Reference" checkbox.
(6) Click the "Save" button.

How to see Home screen​

(1) Select each parameter.

  • Service ID: Limit transaction data to a single service ID. Only service IDs with Payment Reference permission are shown.
  • Currency Code: Limit transaction data to a single currency. Available currencies are updated when selecting a service ID, and contain all currencies that were recently used.
  • Summary Type:
    • Captured: Only include Capture/AuthorizeAndCapture transactions, and Cancel/Modify transactions on captured payments.
    • Authorized: Only include Authorize/AuthorizeAndCapture transactions, and Cancel/Modify transactions on all payments.

The Service ID and Currency Code fields support type-to-search.

(2) Enable to switch sales data period (Month or Day).
*The screenshots on the left show the Month (top) and Daily screens, respectively.
In Day tab, the X-axis of each chart is in 24-hour notation.

(3) Display Gross Sales Amount of current period (This Month / Today) and last period (Last Month/Yesterday).

(4) Display Transaction Count of current period (This Month / Today) and last period (Last Month/Yesterday).
Basically, units are not displayed, but In Japanese, "件" is added.(e.g. 4件)

(5) Display amount chart.
Gross amount for last 31 days / 24 hours (similar for "Transactions")

(6) Display transaction count chart.

(7) When placing the cursor a chart elment, a tooltip with the date and time and the exact amount / transaction count is shown.
The tooltip is synced with both chart.


How to see payment transactions​

You can search and view transactions that have been settled using the API of our payment system. You can also change the settlement amount, confirm sales, and cancel the transaction by selecting the target payment and proceeding to "Payment Details".

(1) Push "Payments" button on the header.
Then you can see "Payments" page.
(2) Choose your service ID you want to use.
(3) Enter the criteria you want to search.
(4) Push "SEARCH" button.
(5) You can see the search result.
(6) Click a payment transaction if you want to see the more details.
