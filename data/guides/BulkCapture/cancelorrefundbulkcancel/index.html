CancelOrRefund (Bulk Cancel)

  • Overview
  • Prerequisite
    • Only saleWithoutAuthorize (card only) or saleWithoutAuthorize (card with point)
  • File Server
  • I/F
    • Request File
    • Response File
  • Tips

Overview​

We support the CancelOrRefund feature.

You can use it when you want to have cancel or refund operations after making payments with Bulk Capture (AuthorizeAndCapture).

Prerequisite​

You have to set the same "Payment ID" as

  • "Payment ID" used in a request file of Bulk Capture (AuthorizeAndCapture)
    AuthorizeAndCapture (Bulk Capture)#INfile
  • "Payment ID" used in Payment Gateway

Only saleWithoutAuthorize (card only) or saleWithoutAuthorize (card with point)​

The card issuer should be Rakuten Card.

However, Bulk Capture doesn't check it and returns success response even if the card issuer isn't Rakuten Card.

I/F​

Request File​

File Name​

{yourServiceId}_yyyyMMdd_cancel.json

The file name has the following meanings.

"yyyyMMdd" will be used as the default value of "saleDate" parameter to be set in request file.
"cancel" shows CancelOrRefund operation as a request type. This is necessary to distinguish the operation from AuthorizeAndCapture.

Example​

all-payportal_20210611_cancel.json

Format​

The Request file must meet all of the following conditions.

  • Maximum number of records in a file is recommended less than 2,000,000.
    • No error would be occurred even it is over 2,000,000, but please make sure to consult with Payment Platform Team in advance if you need to upload a file with over 2,000,000 records.
  • Each request is written per row.
  • No BOM is added.
    • For details about BOM, please see https://en.wikipedia.org/wiki/Byte_order_mark.
  • Each row, including the last line, is terminated with UNIX newline character (LF, "\n", 0x0a).
    • Please pay special attention to the last line break, as it is often forgotten.
