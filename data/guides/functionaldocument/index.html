Fraud Prevention​

Transaction Guard (previously called PayGuard)​

PGW internally connects Transaction Guard, which is a Rakuten in-house fraud prevention platform. If you would like to use it, you can use it at requesting a payment request though PGW APIs.

For example, if you want to enable 3D Secure authentication only with certain conditions, Transaction Guard will help with the decision if you use 3D Secure or not.

Transaction Guard decision pattern with 3D Secure​

Send a payment request with "withThreeDSecure" flag Whether the service is sending 3D secure flag with true in a payment request

Use HTML type API to enable browser redirect

This is about how to validate a credit card by Card Verification before authorization. Card Holder Authentication is executed after authorization and only if the response is success.

  1. You can utilize this feature by setting "0" into the card payment amount card when you call the following payment APIs. Note that there is a possibility you get an error at Card Verification when the issuer rejects the amount 0.
  2. You can utilize this feature by setting a billing address into "custom" as japanBillingAddress when you make a payment request with the following APIs.
